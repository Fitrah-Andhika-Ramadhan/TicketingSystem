The ticketing system is designed to manage customer support tickets with:

ARCHITECTURE
============

DATABASE (Prisma Schema)
- Ticket model: Core ticket data with auto-incrementing ticket numbers
- Comment model: Public comments and internal notes
- Attachment model: File attachments on tickets
- TicketHistory model: Audit trail of all changes
- Notification model: System notifications for ticket updates
- User model: Enhanced with ticket relations

API ENDPOINTS
=============

GET /api/tickets
- List all tickets with optional filters
- Query parameters: status, priority
- Returns: Array of tickets

POST /api/tickets
- Create new ticket
- Required: title, description, category
- Optional: priority (default: MEDIUM)
- Returns: New ticket object with auto-generated ticketNumber

GET /api/tickets/[id]
- Get ticket details with comments and history
- Returns: Complete ticket with comments and history

PATCH /api/tickets/[id]
- Update ticket (status, priority, assignment)
- Supports: status, priority, assignedTo, title, description
- Returns: Updated ticket object

DEPLOYMENT
==========

Before deploying to Vercel:

1. Database Setup
   - Replace mock data with real Prisma queries
   - Run migrations: pnpm run db:push
   - Seed sample data: pnpm run db:seed

2. Environment Variables
   - JWT_SECRET: Generate random 32+ char string
   - DATABASE_URL: PostgreSQL connection string
   - NEXT_PUBLIC_API_URL: Your Vercel deployment URL

3. Build & Deploy
   - pnpm run build
   - Deploy to Vercel: vercel --prod
