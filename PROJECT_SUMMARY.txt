================================================================================
TECHNOLOGY STACK
================================================================================

FRONTEND:
  Framework: Next.js 16 (React 19) with TypeScript
  Styling: Tailwind CSS + shadcn/ui component library
  State: Zustand + SWR (React hooks)
  Charts: Recharts
  Real-time: Socket.io (WebSocket)
  Forms: React Hook Form + Zod validation

BACKEND:
  API: Next.js API Routes
  Language: TypeScript
  ORM: Prisma
  Authentication: JWT + bcrypt
  Validation: Zod schemas
  Logging: Winston/Pino

DATABASE:
  Type: PostgreSQL (cloud-hosted)
  Management: Navicat (GUI tool)
  ORM: Prisma for data access
  Connection Pool: PgBouncer

HOSTING & INFRASTRUCTURE:
  Frontend: Vercel (Edge Functions)
  Backend: Vercel (Serverless Functions)
  Database: PostgreSQL (managed service)
  Storage: Vercel Blob (documents)
  Monitoring: Sentry (error tracking)
  Analytics: PostHog (optional)
  CI/CD: GitHub Actions

DATABASE SCHEMA (12 CORE TABLES)
================================================================================

users
  - id, email, password_hash, full_name, role, department, is_active
  - Manages: Admin, Manager, Supervisor, Finance, Viewer roles

projects
  - id, name, description, location, start_date, planned_end_date, status
  - Tracks main project information and timeline

blocks
  - id, project_id, block_name, unit_count, status, completion_percentage
  - Individual blocks/units with progress tracking

phases
  - id, project_id, phase_name, status, planned_start, actual_start/end
  - Project phases with milestone tracking

milestones
  - id, phase_id, milestone_name, status, planned_date, actual_date
  - Key achievement targets within phases

budget_lines
  - id, project_id, category, allocated_amount, spent_amount
  - Budget tracking by category (materials, labor, equipment, etc.)

documents
  - id, project_id, file_name, file_type (spr/drawing/report), version
  - SPR and technical document storage with versioning

metrics_data
  - id, project_id, metric_type, metric_value, recorded_at, source
  - Real-time sensor and manual input data

photo_documentation
  - id, block_id, image_url, description, taken_at, tags
  - Progress photos with metadata

notifications
  - id, user_id, title, message, type (alert/info/warning), is_read
  - User notifications with delivery tracking

audit_logs
  - id, user_id, action, entity_type, changes, timestamp, ip_address
  - Complete activity audit trail

relationships & supporting tables
  - User-role assignments, permissions, metadata tables, etc.
